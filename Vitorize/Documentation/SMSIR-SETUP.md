# راه‌اندازی سرویس پیامک SMS.ir در ویتورایز

این سند نحوه‌ی راه‌اندازی، پیکربندی و بهره‌برداری از زیرسیستم پیامک ویتورایز مبتنی بر
**SMS.ir** را شرح می‌دهد.

---

## ۱. معماری در یک نگاه

```
کد تجاری / کنترلر / کامپوننت
        │  (فقط از طریق ISmsService)
        ▼
ISmsService  ── SmsService (Infrastructure)
        │            ├─ ISmsSettingsProvider  (کش تنظیمات، خواندن از جدول Settings)
        │            └─ ISmsSender  ── SmsIrSender (آداپتور SDK رسمی IPE.SmsIr)
        ▼
رویدادهای تجاری → ISmsOutboxEnqueuer → جدول OutboxMessages
        ▼
OutboxProcessorBackgroundService → ISmsService → SMS.ir
```

- **هیچ** کنترلر/کامپوننت/سرویس تجاری‌ای مستقیماً `SmsIr` را صدا نمی‌زند.
- تنها نقطه‌ی تماس با SDK، کلاس `SmsIrSender` است.
- کلید API فقط در جدول `Settings` نگهداری می‌شود (نه در `appsettings.json`، نه هارد‌کد).

## ۲. بسته رسمی

- مخزن رسمی: <https://github.com/IPeCompany/SmsPanelV2.dotNet>
- بسته NuGet: **`IPE.SmsIr`** (نسخه نصب‌شده: `1.2.7`) — در پروژه `Vitorize.Infrastructure`.
- متدهای استفاده‌شده: `VerifySendAsync` (قالبی/کد یکبار‌مصرف)، `BulkSendAsync` (متن ساده)،
  `GetCreditAsync` (اعتبار)، `GetLinesAsync` (خطوط).

## ۳. ساخت حساب و کلید API در SMS.ir

1. در <https://sms.ir> ثبت‌نام و احراز هویت کنید.
2. یک **خط اختصاصی** تهیه کنید (برای پیامک متنی/Bulk لازم است؛ برای ارسال قالبی الزامی نیست).
3. از بخش «توسعه‌دهندگان / API»، یک **کلید API** بسازید.
4. **قالب‌های پیامک** موردنیاز را بسازید و منتظر تایید بمانید (جدول قالب‌ها در
   [SMSIR-TEMPLATES.md](SMSIR-TEMPLATES.md)).

## ۴. پیکربندی از پنل ادمین

مسیر: **پنل ادمین › تنظیمات › اعلان‌ها**. بخش «سرویس پیامک SMS.ir» شامل:

| تنظیم | کلید | توضیح |
|------|------|-------|
| فعال‌سازی | `Sms.IsEnabled` | روشن/خاموش کردن کل ارسال پیامک |
| کلید API | `Sms.ApiKey` | محرمانه؛ به‌صورت ماسک‌شده نمایش داده می‌شود |
| شماره خط | `Sms.DefaultLineNumber` | برای پیامک متنی (Bulk) |
| شناسه‌های قالب | `Sms.*TemplateId` | از پنل SMS.ir |
| اعتبار کد | `Sms.OtpExpiryMinutes` | مدت اعتبار کد یکبار‌مصرف |
| فاصله ارسال مجدد | `Sms.OtpResendCooldownSeconds` | |
| حداکثر تلاش | `Sms.OtpMaxAttempts` | |
| سقف روزانه | `Sms.DailyOtpLimitPerMobile` | |
| بازتلاش | `Sms.MaxRetryCount` / `Sms.RetryDelaySeconds` | برای Outbox |
| استفاده از Outbox | `Sms.UseOutbox` | ارسال ناهم‌زمان رویدادهای تجاری |

> **نکته امنیتی:** کلید API و شماره خط هرگز در endpoint عمومی تنظیمات برنمی‌گردند و در پنل
> ادمین به‌صورت `********` نمایش داده می‌شوند. اگر فیلد را دست‌نخورده ذخیره کنید، مقدار واقعی
> بازنویسی **نمی‌شود**؛ برای تغییر، مقدار جدید وارد کنید.

## ۵. اعمال تنظیمات پایگاه‌داده (Database-First)

هیچ مهاجرت EF ساخته نمی‌شود. اسکریپت‌های زیر را به‌ترتیب اجرا کنید:

1. `Database/2026-07-13_seed_sms_settings.sql` — **داده‌ای، idempotent**. کلیدهای `Sms.*` را
   می‌سازد (اگر با اجرای برنامه seed نشده باشند). مقادیر موجود (ازجمله کلید API) حفظ می‌شوند.
2. `Database/2026-07-13_otpcodes_purpose_login_constraint.sql` — **اسکیمایی، دفاعی، idempotent**.
   فقط اگر روی ستون `OtpCodes.Purpose` یک CHECK constraint محدودکننده وجود داشته باشد، آن را
   به‌گونه‌ای بازمی‌سازد که مقدار `4` (Login) را بپذیرد. در پایگاه‌داده‌ی استاندارد بدون‌اثر است.

> Seeder برنامه (`VitorizeSeedService`) نیز همان کلیدها را در استارتاپ می‌سازد؛ اجرای اسکریپت
> داده‌ای اختیاری است و صرفاً امکان provisioning بدون ری‌استارت را می‌دهد.

## ۶. فعال‌سازی و ارسال پیامک آزمایشی

1. `Sms.IsEnabled = true`، `Sms.ApiKey` و شناسه‌ی قالب‌ها را تنظیم کنید.
2. در همان صفحه، بخش «ارسال پیامک آزمایشی»: شماره، قالب و پارامترها را وارد و ارسال کنید.
3. «بررسی وضعیت» اعتبار حساب، خطوط و صحت اتصال را نشان می‌دهد (بدون افشای کلید).

## ۷. رفتار توسعه/تست

- در تست‌ها **هرگز** API واقعی SMS.ir صدا زده نمی‌شود؛ از `FakeSmsSender` استفاده می‌شود.
- اگر `Sms.IsEnabled=false` باشد، هر ارسال با `SmsFailureReason.Disabled` برمی‌گردد و پیامی ارسال نمی‌شود.
- کد یکبار‌مصرف هرگز در Production در کنسول/لاگ چاپ نمی‌شود (رفتار placeholder قبلی حذف شده است).

## ۸. چک‌لیست Production

- [ ] `Sms.IsEnabled = true`
- [ ] `Sms.ApiKey` معتبر تنظیم شده و در لاگ‌ها دیده نمی‌شود
- [ ] `Sms.DefaultLineNumber` (اگر پیامک متنی لازم است)
- [ ] شناسه‌ی همه‌ی قالب‌های استفاده‌شده تنظیم شده و در پنل SMS.ir «تاییدشده» است
- [ ] «بررسی وضعیت» اتصال و اعتبار مثبت نشان می‌دهد
- [ ] `Sms.LogSensitiveData = false`
- [ ] اسکریپت‌های SQL اجرا شده‌اند
- [ ] پشتیبان‌گیری از تنظیمات (کلید API)

## ۹. عیب‌یابی و خطاهای رایج SMS.ir

| نشانه (`SmsFailureReason`) | علت محتمل | راه‌حل |
|---|---|---|
| `Disabled` | `Sms.IsEnabled=false` | فعال کنید |
| `NotConfigured` | کلید API خالی | کلید را تنظیم کنید |
| `Unauthorized` | کلید API نامعتبر | کلید صحیح را وارد کنید |
| `InsufficientCredit` | اعتبار ناکافی | شارژ حساب SMS.ir |
| `InvalidTemplate` | شناسه قالب خالی/اشتباه یا تاییدنشده | شناسه صحیح؛ تایید قالب |
| `InvalidParameter` | نام پارامتر با قالب مطابق نیست | نام‌ها را دقیقاً مطابق قالب کنید |
| `InvalidLineNumber` | خط اختصاصی تنظیم نشده | `Sms.DefaultLineNumber` را تنظیم کنید |
| `TooManyRequests` | فراتر از حد نرخ | کمی صبر کنید |
| `Network` / `Timeout` | مشکل شبکه | اتصال سرور؛ برای رویدادها Outbox بازتلاش می‌کند |
```
